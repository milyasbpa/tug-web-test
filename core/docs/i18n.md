# i18n Setup: next-intl

## Overview

- Using `next-intl` (best fit for Next.js App Router)
- Translation JSON files are managed manually in `core/i18n/json/`

## Checklist

- [x] **1. Set up TypeScript type safety for translation keys**
  - `core/i18n/json/index.ts` is auto-generated by the script (step 3) — imports only from `en/`
  - Only `en` is imported because this file is **for TypeScript type inference only**, not runtime data
  - All locales (`en`, `de`) share the same keys, so TypeScript only needs to know the "shape" from `en`
  - Create `core/i18n/types/i18n.d.ts` to augment `IntlMessages` in next-intl:

    ```ts
    import namespaces from '../json/index';

    type I18nNamespaces = typeof namespaces;

    declare global {
      interface IntlMessages extends I18nNamespaces {}
    }
    ```

  - `interface extends` only accepts a **TypeReference**, so `typeof X` cannot be used directly
  - Therefore `typeof namespaces` must first be assigned to a `type` alias before extending
  - After this, when typing `t('title')` inside the `home` namespace, TypeScript will:
    - ✅ Autocomplete all available keys
    - ✅ Error if the key does not exist in the JSON
  - **No manual type generation needed** — TypeScript reads directly from the JSON files

- [x] **2. Configure next-intl**
  - Create `core/i18n/routing.ts` — defines locales (`en`, `de`) with `en` as the defaultLocale
  - Create `core/i18n/request.ts` — loads messages from `core/i18n/json/[locale]/`
  - Update `next.config.ts` with the next-intl plugin

- [x] **3. Set up middleware**
  - Create `middleware.ts` at the root to handle automatic locale routing
  - `localePrefix: 'as-needed'` in `routing.ts` — the default locale (`en`) uses no URL prefix:

    | URL        | Behavior                                          |
    | ---------- | ------------------------------------------------- |
    | `/blog`    | ✅ Renders, locale `en` (default)                 |
    | `/en/blog` | ↩️ Redirects to `/blog` (default prefix stripped) |
    | `/de/blog` | ✅ Renders, locale `de`                           |

- [x] **4. Restructure app directory**
  - Create `app/[locale]/` folder and move `layout.tsx` & `page.tsx` into it
  - Wrap the layout with `NextIntlClientProvider`
  - `app/layout.tsx` is made a minimal passthrough — `html`, `body`, fonts & providers live in `app/[locale]/layout.tsx`

- [x] **5. Create helpers & usage examples**
  - `core/i18n/navigation.ts` — locale-aware `Link`, `useRouter`, `usePathname`, `redirect`, `getPathname`
  - `core/components/examples/i18n-client.tsx` — example of `useTranslations()` in a client component (`'use client'`)
  - `core/components/examples/i18n-server.tsx` — example of `getTranslations()` in a server component (async)

---

> Translation JSON files live in `core/i18n/json/` and are committed to source control. Edit them directly when adding or updating translations.
